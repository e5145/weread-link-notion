# Notion 页面布局

新版布局目标是：信息完整，但页面不臃肿；首页轻，数据库承载细节。

## 首页

第一次同步会在 `NOTION_PAGE` 指向的页面里追加：

1. `WeRead Link Notion` 标题。
2. 简短说明。
3. 阅读热力图图片。
4. 四个数据库入口。

热力图使用 Notion 原生 image block，图片来自仓库里的 `assets/heatmap.png`。这样比嵌入第三方网页更稳定，也更适合 Notion 桌面端和移动端。

## 数据库

### 书库

保存书架里的可见条目。

包括：

- 电子书
- 有声书
- 文章收藏入口

主要字段：

| 字段 | 说明 |
|---|---|
| `书名` | 页面标题 |
| `Book ID` | 微信读书唯一 ID |
| `类型` | 电子书 / 有声书 / 文章收藏 |
| `作者` | 作者或演播者 |
| `分类` | 分类 |
| `状态` | 未读 / 在读 / 读完 |
| `最近阅读` | 最近阅读或收听日期 |
| `私密` | 是否私密 |
| `微信读书链接` | 微信读书网页阅读链接 |
| `划线数` | 划线数量 |
| `想法数` | 想法/点评数量 |
| `书签数` | 书签数量 |

### 笔记

保存划线和个人想法。

主要字段：

| 字段 | 说明 |
|---|---|
| `内容` | 划线或想法正文 |
| `Note ID` | 稳定唯一 ID |
| `Book ID` | 所属书籍 ID |
| `书名` | 所属书籍 |
| `类型` | 划线 / 想法 |
| `章节` | 章节名 |
| `创建时间` | 创建时间 |
| `位置` | 微信读书 range |
| `链接` | 微信读书网页链接 |

### 每日阅读

保存每天的阅读时间。

主要字段：

| 字段 | 说明 |
|---|---|
| `日期` | 日期标题 |
| `Date` | 日期字段 |
| `阅读时长(秒)` | 原始秒数 |
| `阅读时长(分钟)` | 分钟数，便于 Notion 图表 |
| `Year` | ISO 年 |
| `Month` | 月份 |
| `Week` | ISO 周 |

### 同步快照

每次同步后写入一条记录，方便排查。

主要字段：

| 字段 | 说明 |
|---|---|
| `同步时间` | 本次同步时间 |
| `Status` | success / failure |
| `Books` | 本次书库条目数 |
| `Notes` | 本次笔记条数 |
| `Read Days` | 有阅读记录的天数 |
| `Read Minutes` | 年内累计阅读分钟数 |
| `Heatmap` | 热力图 URL |
| `Message` | 运行信息 |

## 推荐手动视图

Notion 官方 API 目前不适合稳定地跨账号创建复杂视图，所以建议第一次同步后手动添加这些视图：

1. `书库`：Gallery 视图，封面字段用 `封面`，按 `状态` 分组。
2. `笔记`：Table 视图，按 `创建时间` 倒序。
3. `每日阅读`：Calendar 视图，用 `Date` 字段。
4. `每日阅读`：Chart 视图，X 轴用 `Date`，Y 轴用 `阅读时长(分钟)`。
5. `同步快照`：Table 视图，按 `同步时间` 倒序。

这样做的好处是页面加载快，数据库仍然可以自由过滤、排序和可视化。
