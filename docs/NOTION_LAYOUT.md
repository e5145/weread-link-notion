# Notion 页面布局说明

新版布局目标是：

- 首页轻，不堆太多块，打开更流畅。
- 高频内容先看到，完整数据留在数据库里。
- 自动同步负责基础结构和图表生成，后续同步只更新数据，不破坏你在 Notion 里新增的视图。

## 首页会出现什么

第一次同步成功后，`NOTION_PAGE` 指向的页面会追加这些内容：

1. 一个蓝色的 `微信读书阅读面板`。
2. 顶部自定义句子，默认是 `看书真好`。
3. 年度摘要：书库数量、笔记数量、阅读日、连续阅读天数和累计分钟数。
4. 阅读热力图图片。
5. 最近阅读和当月阅读时长分布图。
6. 最近笔记、最近划线、推荐好书和阅读画像。
7. 一组完整数据库入口。

五个数据库分别是：

```text
书库
笔记
每日阅读
推荐好书
同步快照
```

图表使用 Notion 原生 image block。图片来自仓库里的：

```text
assets/heatmap.png
assets/monthly-reading.png
assets/reading-profile.png
```

`微信读书阅读面板` 会在每次运行时自动刷新。它不是完整数据库，只是为了让你打开首页时立刻看到关键摘要。完整内容仍然保存在下面五个数据库里。

如果页面之前因为失败调试出现了多组 `书库 / 笔记 / 每日阅读 / 推荐好书 / 同步快照`，同步器会保留每类数据库里数据最多的那个，其余同名数据库入口会自动归档。

## 书库

保存微信读书书架里的条目，包括电子书、有声书和文章收藏入口。

| 字段 | 说明 |
|---|---|
| `书名` | Notion 页面标题 |
| `Book ID` | 微信读书唯一 ID |
| `类型` | 电子书 / 有声书 / 文章收藏 |
| `作者` | 作者或演播者 |
| `分类` | 微信读书分类 |
| `状态` | 未读 / 在读 / 读完 |
| `最近阅读` | 最近阅读或收听日期 |
| `微信读书链接` | 微信读书网页链接 |
| `划线数` | 这本书的划线数量 |
| `想法数` | 这本书的想法或点评数量 |
| `书签数` | 这本书的书签数量 |

## 笔记

保存划线和个人想法。

| 字段 | 说明 |
|---|---|
| `内容` | 划线文本或想法正文 |
| `Note ID` | 稳定唯一 ID |
| `Book ID` | 所属书籍 ID |
| `书名` | 所属书籍 |
| `类型` | 划线 / 想法 |
| `章节` | 章节名 |
| `创建时间` | 笔记创建时间 |
| `位置` | 微信读书 range |
| `链接` | 微信读书网页链接 |

## 每日阅读

保存每天的阅读时长。当月阅读时长分布和年度热力图都基于这份数据生成。

| 字段 | 说明 |
|---|---|
| `日期` | 页面标题 |
| `Date` | Notion 日期字段 |
| `阅读时长(秒)` | 原始秒数 |
| `阅读时长(分钟)` | 分钟数，方便 Notion 图表 |
| `Year` | ISO 年 |
| `Month` | 月份 |
| `Week` | ISO 周 |

## 推荐好书

保存微信读书 Skill 的个性化推荐结果，每次同步都会更新。

| 字段 | 说明 |
|---|---|
| `书名` | Notion 页面标题 |
| `Book ID` | 微信读书唯一 ID |
| `作者` | 作者 |
| `分类` | 微信读书分类 |
| `推荐理由` | 微信读书返回的推荐理由 |
| `评分` | 评分，按 10 分制展示 |
| `评分人数` | 评分人数 |
| `评分标签` | 例如“神作”“力荐”等 |
| `在读人数` | 在读人数 |
| `微信读书链接` | 打开书籍的链接 |

## 同步快照

每次同步后写入一条记录，方便排错和观察趋势。

| 字段 | 说明 |
|---|---|
| `同步时间` | 本次同步时间 |
| `Status` | success / failure |
| `Books` | 本次书库条目数 |
| `Notes` | 本次笔记条数 |
| `Read Days` | 有阅读记录的天数 |
| `Read Minutes` | 年内累计阅读分钟数 |
| `Recommendations` | 本次推荐书数量 |
| `Streak Days` | 当前连续阅读天数 |
| `Heatmap` | 热力图 URL |
| `Reading Profile` | 阅读画像 URL |
| `Monthly Chart` | 当月阅读时长分布 URL |
| `Message` | 运行信息 |

## 视图建议

Notion 官方 API 更适合创建数据库和写入数据，不适合稳定地跨账号创建复杂视图、图表样式和个人偏好的布局。

建议你同步成功后，在 Notion 里自己新增这些视图：

- `书库`：Gallery 按 `状态` 分组，Table 按 `最近阅读` 倒序。
- `笔记`：Table 按 `创建时间` 倒序，Filter 只看 `类型 = 想法`。
- `每日阅读`：Calendar 用 `Date` 字段，Chart 用 `阅读时长(分钟)`。
- `推荐好书`：Gallery 展示封面，Table 按 `评分` 或 `在读人数` 排序。
- `同步快照`：Table 按 `同步时间` 倒序，只看最近几次运行。
